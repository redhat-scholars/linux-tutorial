Installing unpackaged software

In some cases, you’ll need to install an application from it’s source code. Let’s take a look at how to compile and install new software using RHEL.

Downloading the source code

While there are plenty of applications we could install, let’s take Vim for example. Vim is a popular text editor used in most Linux systems, and is simple to use. First, let’s go to the release page for Vim, and download the most recent tar.gz, which contains the software in a compressed tar format.

Right-click and copy the link address for this specific release so we can then download it in our RHEL machine. Next, let’s use curl to download this specific tar.gz link to our local machine.

curl -OL https://github.com/vim/vim/archive/refs/tags/v9.0.0652.tar.gz

After a few seconds, we’ll see that the file has been downloaded in our current directory.

ls
Downloads  file1.txt  file2.txt  v9.0.0652.tar.gz

Extracting the archive

To extract the source code from the package, we can use the tar command with a few arguments like so.

tar xvf v9.0.0652.tar.gz

After everything has been extracted, you’ll now have a new file in your local directory, named v9.0.0652 or the respective name of the downloaded tar file. Let’s do a quick cd command to change directories into this new folder, and list the items in our directory.

cd vim-9.0.0652
ls

CONTRIBUTING.md  Makefile  READMEdir  README_VIM9.md  tools    vimtutor.com

Installing the source code

Now that we’re in the newly extracted folder, we can install the unpackaged software. Firstly, however, we need to install the development tools, which include gcc, our C compiler, and more. This can be done by using dnf as we have done before, with the following command. You’ll also need a terminal library, for this we can use ncurses.

dnf groupinstall "Development Tools" && dnf install ncurses-devel

Now, we can run the configure script in the folder, using the following command. This will analyze the system, and produce a Makefile that contains the logic to build and install the package.

./configure
configure: loading cache auto/config.cache
checking whether make sets $(MAKE)... yes
checking for gcc... gcc
checking whether the C compiler works... yes
...
config.status: creating auto/config.h

Now, we can compile the source code to generate the binaries using the make command.

make
Starting make in the src directory.
...
make[1]: Leaving directory '/home/ec2-user/vim-9.0.0652/src'

Almost there! After make completes, the binary files are ready to install.

make install
Starting make in the src directory.
...
make[1]: Leaving directory '/home/ec2-user/vim-9.0.0652/src'

Congratulations! vim should now be installed properly on your system, and you can double check the installation with the following command.

vim --version
